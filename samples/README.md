## 数据格式：Input → Output

每条 turn 包含两部分：

### Input（模型输入 / 标注前的数据）

```json
"input": {
  "text_asr":          "喂，哎你好，王总，嗯，我是上午跟你啊，我是上午给你打电话的，做记账报税的那个。",
  "wav_path":          "data/S0000001/A0000001.wav",
  "auto_emotion_hint": "中立"
}
```

| 字段 | 说明 |
|------|------|
| `text_asr` | FunASR 自动转写文本（可能有错误） |
| `wav_path` | 音频切片路径（相对于数据包根目录） |
| `auto_emotion_hint` | 系统自动预测的情感（仅供参考，需人工核对） |

### Output（标注结果）

```json
"output": {
  "transcript_corrected": "喂，哎你好，王总，嗯，我是上午跟你啊，我是上午给你打电话的，做记账报税的那个。诶。嗯这会儿有时间，不",
  "emotion":              "4-中立",
  "sales_strategy":       "开场白/自我介绍",
  "customer_intent":      "N/A-销售turn",
  "paralinguistic":       ["无"],
  "notes":                ""
}
```

| 字段 | 说明 | 取值 |
|------|------|------|
| `transcript_corrected` | 修正后的转写文本 | 字符串 |
| `emotion` | 情感标签 | 0-生气 / 1-厌恶 / 2-恐惧 / 3-高兴 / 4-中立 / 5-其他 / 6-伤心 / 7-惊讶 / 8-未知 |
| `sales_strategy` | 销售策略 | 开场白 / 需求挖掘 / 产品介绍 / 异议处理 / 逼单促成 / 价格谈判 / 关系维护 / 收尾挂断 / N/A-客户turn |
| `customer_intent` | 客户意图 | 有兴趣 / 中性观望 / 软拒绝 / 硬拒绝 / 提问了解 / 讨价还价 / 无关回应 / N/A-销售turn |
| `paralinguistic` | 副语言事件（多选） | 笑声 / 叹气 / 犹豫吞吐 / 填充词 / 语气词 / 清嗓 / 拖音 / 惊声 / 无 |
| `notes` | 备注 | 自由文本 |

## 标注要点（从此通话总结）

**ASR 修正常见类型：**
- 同音字替换：`干嘛账` → `伽马账`，`报废税` → `报税`
- 说话人边界错误：销售句末的客户短回应被并入（在备注中标注）
- 填充词误识：`哦` / `诶` / `嗯` 有时被吞掉或替换

**情感分布（此通话）：**
- 中立：25/35（71%）— 正常商业通话
- 高兴：4/35 — 客户表现出兴趣的时刻
- 生气：2/35 — 客户短暂不耐烦

**Diarization 问题（已修正）：**
- 约 37% 的 turn 存在说话人边界不精确（短回应被并入上一个 turn）
- F0 校验修正了 0 条（此通话为同性别对话，F0 差异不足 30Hz，跳过）
